Cyclus Archetypes¶

This is a collection of archetypes found in the agents module that comes with Cyclus itself.

:agents:Source¶

A minimum implementation source facility that provides a commodity with a given capacity

entity:facility
parents:[cyclus::Facility]
all_parents:[cyclus::Agent, cyclus::Facility, cyclus::Ider, cyclus::StateWrangler, cyclus::TimeListener, cyclus::Trader]
name:cyclus::Source

State Variables:

commod:

std::string

source commodity

commodity that the source facility supplies

uitype:outcommodity
schematype:token
recipe_name:

std::string, default=

commodity recipe name

Recipe name for source facility’s commodity.If empty, source supplies material with requested compositions.

uitype:recipe
schematype:token
capacity:

double

source capacity

amount of commodity that can be supplied at each time step

Schema:

<interleave>
    <element name="commod">
        <data type="token"/>
    </element>
    <optional>
        <element name="recipe_name">
            <data type="token"/>
        </element>
    </optional>
    <element name="capacity">
        <data type="double"/>
    </element>
</interleave>

:agents:Sink¶

A minimum implementation sink facility that accepts specified amounts of commodities from other agents

entity:facility
parents:[cyclus::Facility]
all_parents:[cyclus::Agent, cyclus::Facility, cyclus::Ider, cyclus::StateWrangler, cyclus::TimeListener, cyclus::Trader]
name:cyclus::Sink

State Variables:

recipe_name:

std::string, default=

input/request recipe name

Name of recipe to request.If empty, sink requests material no particular composition.

uitype:recipe
in_commods:

std::vector<std::string>

input commodities for the sink

commodities that the sink facility accepts

uitype:[u’oneormore’, u’incommodity’]
capacity:

double

sink capacity

capacity the sink facility can accept at each time step
max_inv_size:

double, default=1e+299

sink maximum inventory size

total maximum inventory size of sink facility

Schema:

<interleave>
    <optional>
        <element name="recipe_name">
            <data type="string"/>
        </element>
    </optional>
    <element name="in_commods">
        <oneOrMore>
            <element name="val">
                <data type="string"/>
            </element>
        </oneOrMore>
    </element>
    <element name="capacity">
        <data type="double"/>
    </element>
    <optional>
        <element name="max_inv_size">
            <data type="double"/>
        </element>
    </optional>
</interleave>

:agents:NullRegion¶

A region that owns the simulation’s institutions but exhibits null behavior. No parameters are given when using the null region.

entity:region
parents:[cyclus::Region]
all_parents:[cyclus::Agent, cyclus::Ider, cyclus::Region, cyclus::StateWrangler, cyclus::TimeListener]
name:cyclus::NullRegion

Schema:

<text/>

:agents:NullInst¶

An instition that owns facilities in the simulation but exhibits null behavior. No parameters are given when using the null institution.

entity:institution
parents:[cyclus::Institution]
all_parents:[cyclus::Agent, cyclus::Ider, cyclus::Institution, cyclus::StateWrangler, cyclus::TimeListener]
name:cyclus::NullInst

Schema:

<text/>

:agents:KFacility¶

A facility designed for integration tests that both provides and consumes commodities. It changes its request and offer amounts based on a power law with respect to time.

entity:facility
parents:[cyclus::Facility]
all_parents:[cyclus::Agent, cyclus::Facility, cyclus::Ider, cyclus::StateWrangler, cyclus::TimeListener, cyclus::Trader]
name:cyclus::KFacility

State Variables:

in_commod:

std::string

input commodity

commodity that the k-facility consumes

uitype:incommodity
schematype:token
out_commod:

std::string

output commodity

commodity that the k-facility supplies

uitype:outcommodity
schematype:token
recipe_name:

std::string, shape=[50]

in-commodity recipe name

recipe name for the k-facility’s in-commodity

uitype:recipe
schematype:token
in_capacity:

double

input commodity capacity

number of commodity units that can be taken at each time step (infinite capacity can be represented by a very large number
out_capacity:

double

output commodity capacity

number of commodity units that can be supplied at each time step (infinite capacity can be represented by a very large number
current_capacity:
 

double, default=0

current output capacity

number of output commodity units that can be supplied at the current time step (infinite capacity can be represented by a very large number
max_inv_size:

double, default=1e+299

k-facility maximum inventory size

total maximum inventory size of the k-facility
k_factor_in:

double

input k-factor

conversion factor that governs the behavior of the k-facility’s input commodity capacity
k_factor_out:

double

output k-factor

conversion factor that governs the behavior of the k-facility’s output commodity capacity

Schema:

<interleave>
    <element name="in_commod">
        <data type="token"/>
    </element>
    <element name="out_commod">
        <data type="token"/>
    </element>
    <element name="recipe_name">
        <data type="token"/>
    </element>
    <element name="in_capacity">
        <data type="double"/>
    </element>
    <element name="out_capacity">
        <data type="double"/>
    </element>
    <optional>
        <element name="current_capacity">
            <data type="double"/>
        </element>
    </optional>
    <optional>
        <element name="max_inv_size">
            <data type="double"/>
        </element>
    </optional>
    <element name="k_factor_in">
        <data type="double"/>
    </element>
    <element name="k_factor_out">
        <data type="double"/>
    </element>
</interleave>

:agents:Predator¶

A facility that represents predators in the Lotka-Volterra integration tests

entity:facility
parents:[cyclus::Facility]
all_parents:[cyclus::Agent, cyclus::Facility, cyclus::Ider, cyclus::StateWrangler, cyclus::TimeListener, cyclus::Trader]
name:cyclus::Predator

State Variables:

commod:

std::string

predator commodity

commodity that the predator supplies

uitype:outcommodity
schematype:token
prey:

std::string

predator’s prey

prey that the predator hunts

uitype:incommodity
schematype:token
full:

double, default=1

feast size

how many units of prey a predator consumes until it is satisfied
hunt_cap:

double, default=1

hunting yield

how many units of prey a predator can catch during a hunt
hunt_freq:

int, default=1

hunting frequency

how often a predator needs to hunt
success:

double, default=1

hunting success fraction

fraction of hunting success on a scale from 0 to 1
nchildren:

double, default=1

number of children

number of predator children born at each birthing instance
age:

int, default=0

predator age

age of predator at beginning of simulation
lifespan:

int, default=1

predator lifespan

how long a predator lives
dead:

bool, default=0

dead?

flag for whether predator is currently dead
hunt_factor:

bool, default=0

hunting success factor

whether or not to base hunting success on relative predator/prey populations
birth_and_death:
 

bool, default=0

simultaneous birth and death?

whether or not simultaneous birth and and death are allowed (i.e., can a facility give birth and die in the same time step?)
consumed:

double, default=0

prey consumed

how many units of prey consumed per time step

Schema:

<interleave>
    <element name="commod">
        <data type="token"/>
    </element>
    <element name="prey">
        <data type="token"/>
    </element>
    <optional>
        <element name="full">
            <data type="double"/>
        </element>
    </optional>
    <optional>
        <element name="hunt_cap">
            <data type="double"/>
        </element>
    </optional>
    <optional>
        <element name="hunt_freq">
            <data type="int"/>
        </element>
    </optional>
    <optional>
        <element name="success">
            <data type="double"/>
        </element>
    </optional>
    <optional>
        <element name="nchildren">
            <data type="double"/>
        </element>
    </optional>
    <optional>
        <element name="age">
            <data type="int"/>
        </element>
    </optional>
    <optional>
        <element name="lifespan">
            <data type="int"/>
        </element>
    </optional>
    <optional>
        <element name="dead">
            <data type="boolean"/>
        </element>
    </optional>
    <optional>
        <element name="hunt_factor">
            <data type="boolean"/>
        </element>
    </optional>
    <optional>
        <element name="birth_and_death">
            <data type="boolean"/>
        </element>
    </optional>
    <optional>
        <element name="consumed">
            <data type="double"/>
        </element>
    </optional>
</interleave>
