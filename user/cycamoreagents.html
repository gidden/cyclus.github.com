Cycamore Archetypes¶

This is a collection of archetypes found in the cycamore module that comes with cycamore itself.

:cycamore:Source¶

A source facility that provides a commodity with a given capacity

entity:facility
parents:[cyclus::Facility, cyclus::toolkit::CommodityProducer]
all_parents:[cyclus::Agent, cyclus::Facility, cyclus::Ider, cyclus::StateWrangler, cyclus::TimeListener, cyclus::Trader, cyclus::toolkit::AgentManaged, cyclus::toolkit::CommodityProducer]
name:cycamore::Source

State Variables:

out_commod:

std::string

source output commodity

output commodity that the source facility supplies

uitype:outcommodity
recipe_name:

std::string

commodity recipe name

recipe name for source facility’s commodity

uitype:recipe
capacity:

double, default=1e+299

source capacity

amount of commodity that can be supplied at each time step
current_capacity:
 

double

derived_init:current_capacity = capacity;

Schema:

<interleave>
<element name="out_commod">
    <data type="string" />
</element>
<element name="recipe_name">
    <data type="string" />
</element>
<optional>
    <element name="capacity">
        <data type="double" />
    </element>
</optional>
</interleave>

:cycamore:Sink¶

A sink facility that accepts specified amounts of commodities from other agents

entity:facility
parents:[cyclus::Facility]
all_parents:[cyclus::Agent, cyclus::Facility, cyclus::Ider, cyclus::StateWrangler, cyclus::TimeListener, cyclus::Trader]
name:cycamore::Sink

State Variables:

in_commods:

std::vector<std::string>

input commodities

commodities that the sink facility accepts

uitype:[u’oneormore’, u’incommodity’]
recipe_name:

std::string, default=

requested composition

name of recipe to use for material requests
social_behav:

std::string, default=None

social behavior

type of social behavior used in trade decisions: None, Every, Random where behav_interval describes the time interval for behavior action
behav_interval:

double, default=0

interval for behavior

interval of social behavior: Every or EveryRandom. If 0 then behavior is not implemented
user_pref:

int, default=0

user-defined preference

change the default preference for requests from this agent
avg_qty:

double, default=1e+299

sink avg_qty

mean for the normal distribution that is sampled to determine the amount of material actually requested at each time step
sigma:

double, default=0

standard deviation

standard deviation (FWHM) of the normal distribution used to generate requested amount of material (avg_qty)
max_inv_size:

double, default=1e+299

sink maximum inventory size

total maximum inventory size of sink facility
inventory:

cyclus::toolkit::ResourceBuff

capacity:max_inv_size

Schema:

<interleave>
<element name="in_commods">
    <oneOrMore>
        <element name="val">
            <data type="string" />
        </element>
    </oneOrMore>
</element>
<optional>
    <element name="recipe_name">
        <data type="string" />
    </element>
</optional>
<optional>
    <element name="social_behav">
        <data type="string" />
    </element>
</optional>
<optional>
    <element name="behav_interval">
        <data type="double" />
    </element>
</optional>
<optional>
    <element name="user_pref">
        <data type="int" />
    </element>
</optional>
<optional>
    <element name="avg_qty">
        <data type="double" />
    </element>
</optional>
<optional>
    <element name="sigma">
        <data type="double" />
    </element>
</optional>
<optional>
    <element name="max_inv_size">
        <data type="double" />
    </element>
</optional>
</interleave>

:cycamore:DeployInst¶

An institution that owns, operates, and deploys facilities manually defined in the input file.

entity:institution
parents:[cyclus::Institution]
all_parents:[cyclus::Agent, cyclus::Ider, cyclus::Institution, cyclus::StateWrangler, cyclus::TimeListener]
name:cycamore::DeployInst

Schema:

<oneOrMore>                               
<element name="buildorder">            
  <element name="prototype">           
    <data type="string"/>              
  </element>                             
  <element name="number">              
    <data type="nonNegativeInteger"/>  
  </element>                             
  <element name="date">                
    <data type="nonNegativeInteger"/>  
  </element>                             
</element>                               
</oneOrMore>

:cycamore:ManagerInst¶

An institution that owns and operates a manually entered list of facilities in the input file

entity:institution
parents:[cyclus::Institution, cyclus::toolkit::Builder, cyclus::toolkit::CommodityProducerManager]
all_parents:[cyclus::Agent, cyclus::Ider, cyclus::Institution, cyclus::StateWrangler, cyclus::TimeListener, cyclus::toolkit::AgentManaged, cyclus::toolkit::Builder, cyclus::toolkit::CommodityProducerManager]
name:cycamore::ManagerInst

State Variables:

prototypes:

std::vector<std::string>

facility prototypes

a facility to be managed by the institution

uitype:[u’none’, u’prototype’]

Schema:

<interleave>
<element name="prototypes">
    <oneOrMore>
        <element name="val">
            <data type="string" />
        </element>
    </oneOrMore>
</element>
</interleave>

:cycamore:GrowthRegion¶

A region that governs a scenario in which there is growth in demand for a commodity.

entity:region
parents:[cyclus::Region]
all_parents:[cyclus::Agent, cyclus::Ider, cyclus::Region, cyclus::StateWrangler, cyclus::TimeListener]
name:cycamore::GrowthRegion

State Variables:

commodity_name:

std::string

commodity in demand

name of the commodity experiencing a growth in demand

uitype:commodity
demand_types:

std::vector<std::string>

demand type

mathematical description of demand growth (i.e., linear, exponential, piecewise)
demand_params:

std::vector<std::string>

demand parameters

parameters that define the behavior of the demand type function
demand_times:

std::vector<int>

demand times

vector describing the length of times regarding the piecewise demand type

Schema:

<interleave>
<element name="commodity_name">
    <data type="string" />
</element>
<element name="demand_types">
    <oneOrMore>
        <element name="val">
            <data type="string" />
        </element>
    </oneOrMore>
</element>
<element name="demand_params">
    <oneOrMore>
        <element name="val">
            <data type="string" />
        </element>
    </oneOrMore>
</element>
<element name="demand_times">
    <oneOrMore>
        <element name="val">
            <data type="int" />
        </element>
    </oneOrMore>
</element>
</interleave>

:cycamore:EnrichmentFacility¶

An enrichment facility that intakes a commodity (usually natural uranium) and supplies a user-specified enriched product based on SWU capacity

entity:facility
parents:[cyclus::Facility]
all_parents:[cyclus::Agent, cyclus::Facility, cyclus::Ider, cyclus::StateWrangler, cyclus::TimeListener, cyclus::Trader]
name:cycamore::EnrichmentFacility
niche:enrichment

State Variables:

in_commod:

std::string

input commodity

commodity that the enrichment facility accepts

uitype:incommodity
out_commods:

std::vector<std::string>

output commodities

commodities that the enrichment facility supplies
in_recipe:

std::string

input commodity recipe

recipe for enrichment facility’s input commodity

uitype:recipe
tails_assay:

double, default=0.03

tails assay

tails assay from the enrichment process
swu_capacity:

double, default=1e+299

SWU capacity

separative work unit (SWU) capcity of enrichment facility
max_inv_size:

double, default=1e+299

maximum inventory size

maximum inventory capacity of natural uranium in the enrichment facility
initial_reserves:
 

double, default=0

initial uranium reserves

amount of natural uranium stored at the enrichment facility at the beginning of the simulation
social_behav:

bool, default=0

social behavior

if set to 1 then enable social behavior in trade decisions
current_swu_capacity:
 

double

derived_init:current_swu_capacity = swu_capacity;
feed_assay:

double

feed assay

feed assay for the enrichment process

derived_init:cyclus::Material::Ptr feed = cyclus::Material::CreateUntracked(0, context()->GetRecipe(in_recipe)); feed_assay = cyclus::toolkit::UraniumAssay(feed);
inventory:

cyclus::toolkit::ResourceBuff

capacity:max_inv_size
tails:

cyclus::toolkit::ResourceBuff

Schema:

<interleave>
<element name="in_commod">
    <data type="string" />
</element>
<element name="out_commods">
    <oneOrMore>
        <element name="val">
            <data type="string" />
        </element>
    </oneOrMore>
</element>
<element name="in_recipe">
    <data type="string" />
</element>
<optional>
    <element name="tails_assay">
        <data type="double" />
    </element>
</optional>
<optional>
    <element name="swu_capacity">
        <data type="double" />
    </element>
</optional>
<optional>
    <element name="max_inv_size">
        <data type="double" />
    </element>
</optional>
<optional>
    <element name="initial_reserves">
        <data type="double" />
    </element>
</optional>
<optional>
    <element name="social_behav">
        <data type="boolean" />
    </element>
</optional>
</interleave>
